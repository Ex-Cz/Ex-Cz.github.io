精力管理：第一章--你一定要全情投入

要点：

  • 管理精力，而非管理时间
  • 个人精力得到充分管理后，才能维护团队精力
  • 全情投入才能确保表现和精力状态最优
  • 明确全情投入四原则，依照管理精力三步走

全情投入四原则：

一：全情投入需要调动四种独立且相关联的精力源：体能、情感、思维、意志

按精力和情绪组成坐标平面，只有右上角的高精力、正面情绪才能让自己进入全情投入状态。

二：不时更新精力以平衡消耗

体能、情感、思维、意志四种精力源都各自有自身的 力量、耐力、灵活性和恢复力，全情投入需要调动4*4=16层不同的特征才能实现。
这意味着，要 使用多线程的方式生活，减少单一特征的过度消耗以致力竭崩溃。
因此，我们要学会如何有节奏地消耗和更新精力，保持生命的跃动。

最丰富、最快乐和最高产的生命的共通之处，是全情应对眼前的挑战，同时能够间断地放松，留给精力再生的空间。

学会成为短跑运动员，而非马拉松、长跑：一些时候全情投入，一些时候闲云野鹤。

三：为了提高能力，须突破自己的惯常极限：模仿运动员进行系统训练

我们锻炼情感、思维和意志能力，采用的是与锻炼体能相同的方法

通过系统性地训练，突破自身极限，短暂的疲劳和伤痛后长期地提升自身上限。

尼采：打不倒我们的会让我们变得更强

四：保持积极的精力仪式习惯，即有细致具体的精力管理方法

仪式习惯是指定义明确、具有高度计划性的行为，会自动把人们拉向某条轨道。

变革是艰难的。我们都是习惯的产物，大多数行为都是非自觉和潜意识的，每天重复上一天的行为。如果某件事你每次做之前都需要思考，你很可能不会长久坚持这些事。

让积极仪式习惯成为自然。

三个步骤管理精力

一，明确目标

思考自己更看重什么，决定人现在直至未来各个阶段想要自己变成什么样。

二，面对事实

反思自己以前和当前精力管理的状况，真实地坦白自己的不足，或通过数据客观地反应缺陷。
可以进行多种测试，对自身的体能、情感、思维、意志精力有详尽的了解。

三，付出行动

行动是为了建立长久的精力仪式习惯，而不是寻求短期应急的解决方法。
认识到创造并维持全方位的精力充沛是可行的，我们并非只能被动等待精力随着时间衰退下去。

如何使用github搭建一个博客——个人记录

此文章仅简要记录搭建博客的流程
因在本人电脑（windows）上没有问题，所以针对不兼容的问题，不会给出问题的解决方案，请自行百度
基本上是从b站 卷二兔 那里学的，想看完整视频流程的请移步

1.环境配置 git & nodejs & npm

a.安装git在任意文件夹，建议在C或硬盘中，主要使用其中的 git bash，也可以在下载后右键目标文件夹位置打开。安装后用git --version命令在cmd里检查版本即可。

b.安装nodejs，建议下载最新版本（current）的.msi文件，安装即可。用node -v检查版本即可。

c.安装npm，在bash里输入npm install

2.Hexo安装和搭建

在bash里输入npm install hexo -g，为电脑全局下载hexo
测试是否安装成功hexo -v，出现一大长列即可。

选择一个文件夹存放你的博客组（记为U），可以在文件夹里使用bash的hexo init [文件夹名称]创建一个博客网页用文件夹（记为F），也可以先创建再在其中使用bash的hexo init

注：若无法在文件夹中打开bash，也可以先在下载文件夹里打开bash，再使用cd xx位置来依次进入目标文件夹

生成静态网页hexo g
预览网页hexo s，ctrl+左键点开localhost后在后面加上:4000

3.向github部署

进入github.com（可能需要梯子或加速器），创建账号，在右上角点加号创建一个repository，名称写用户名.github.io，保持public公开，后面不用管直接创建。

在博客用文件夹（F）里用bash输入git config --global user.name '你的用户名'，回车后再输入git config --global user.email '你的注册邮箱'，注意这两个信息都是你注册github时使用的

再检查是否有ssh密钥cd ~/.ssh，若没有，建立ssh-keygen -t rsa -C '你的邮箱'，获取内容cd ~/.ssh再cat id_rsa.pub复制包含ssh-key的所有内容上传至github个人设置的ssh密钥页面
